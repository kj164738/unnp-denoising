# UNNP / Deep Image Prior 图像去噪实验

这个项目实现了一个教学版的 UNNP / Deep Image Prior 图像去噪流程。它不使用外部训练集，而是用固定随机输入驱动一个随机初始化的卷积网络，让网络去拟合带噪图片，并通过早停选择较好的去噪结果。

## 已完成内容

- 使用 PyTorch 实现了 U-Net 风格的 Encoder-Decoder 网络，带跳跃连接。
- 使用 5 张 `skimage.data` 标准示例图，统一处理为 64×64 灰度图，方便 CPU 跑通。
- 对每张图测试 3 个高斯噪声强度：σ=15、25、50。
- 记录 UNNP 的最佳早停点、PSNR 和 SSIM。
- 使用 BM3D 作为传统去噪方法对比。
- 生成结果表格、早停曲线和四联对比图。

## 重要说明

本项目的训练过程没有使用干净图。训练时，网络只拿自己的输出和带噪图比较，并根据这个差异更新网络。

但是，本项目报告里的“最佳早停点”是用干净图评估出来的理论最优点。也就是说，每隔一段训练步数，会拿当前输出和干净图计算一次 PSNR，然后选分数最高的那一步。这适合教学实验，用来证明早停现象存在；但在真实去噪场景里，通常没有干净图可看，所以这不是一个可以直接部署使用的停止规则。

最佳步数是在固定检查间隔里选出来的。本次默认每 25 步评估一次，所以表格里的“最佳步数”是这些检查点中的最佳值，真正的最优点可能落在两次检查之间。

`moon` 这张图比较特殊：画面大面积接近平坦，内容很简单，因此网络很早就能得到较高分数。它适合作为一个特例观察，但不适合作为早停现象的主要证明。主要早停展示图使用的是 `camera, σ=25`。

本次实验使用 64×64 小图，是为了在 CPU 上完整跑通。这样的结果适合课程展示和方法验证，但不能直接和标准分辨率论文结果做严格对比。

不同机器、不同 PyTorch 版本重新运行时，最佳步数和 PSNR 可能有小幅波动。早停曲线的总体趋势更重要，不应把某一次运行的具体步数当成绝对值。本次记录环境为 CPU，PyTorch 版本为 `2.9.1+cpu`。

低噪声实验（例如 σ=15）有时在限定步数内不一定都能观察到明显下降，这通常表示训练还没有进入明显过拟合阶段，不能单独用这类曲线来证明早停。

## 如何运行

当前文件夹已经包含 BM3D 本地依赖目录 `vendor/`。在本机运行：

```bash
python run_experiments.py --output-dir results
```

默认会运行 5 张图 × 3 个噪声强度的完整实验。当前机器检测到的是 CPU 环境，所以默认使用小尺寸图片。如果想跑更大的图或更多步数，可以调整：

```bash
python run_experiments.py --image-size 96 --steps 1400 --demo-steps 2400 --output-dir results_large
```

## 本次实测摘要

运行环境：CPU。PyTorch：`2.9.1+cpu`。BM3D：PyPI `bm3d==4.0.3`，本项目放在 `vendor/` 中作为本地依赖。图片大小：64×64。普通实验 900 步，早停演示 1600 步，每 25 步评估一次。

| 图片 | 噪声 σ | 带噪 PSNR | UNNP 最佳 PSNR | UNNP 最佳步数 | BM3D PSNR |
|---|---:|---:|---:|---:|---:|
| camera | 15 | 24.71 | 28.27 | 900 | 31.66 |
| camera | 25 | 20.49 | 26.86 | 850 | 28.85 |
| camera | 50 | 15.16 | 23.74 | 450 | 24.93 |
| astronaut | 15 | 24.88 | 26.82 | 900 | 28.98 |
| astronaut | 25 | 20.60 | 24.32 | 650 | 25.50 |
| astronaut | 50 | 15.14 | 20.43 | 525 | 21.04 |
| coffee | 15 | 24.80 | 29.14 | 650 | 30.80 |
| coffee | 25 | 20.63 | 26.42 | 775 | 28.11 |
| coffee | 50 | 15.07 | 22.15 | 375 | 23.38 |
| coins | 15 | 24.66 | 29.86 | 400 | 29.09 |
| coins | 25 | 20.44 | 26.76 | 400 | 26.91 |
| coins | 50 | 14.86 | 22.70 | 250 | 23.13 |
| moon | 15 | 24.70 | 31.74 | 500 | 35.53 |
| moon | 25 | 20.25 | 27.12 | 50 | 33.29 |
| moon | 50 | 14.28 | 27.26 | 75 | 29.21 |

检查结果：15 组实验中，UNNP 的最佳结果都高于对应带噪图。`camera, σ=25` 的早停演示峰值在第 850 步，PSNR 为 26.86 dB，曲线中可以看到峰值后回落。这说明在这组有代表性的演示实验里，网络后期开始拟合噪声，因此需要在合适时机停止；在低噪声或内容很简单的图上，这种回落不一定同样明显。

## 代码结构

- `run_experiments.py`：主入口，负责完整实验流程。
- `src/unnp_denoise/data.py`：测试图加载和加噪。
- `src/unnp_denoise/model.py`：UNNP / DIP 网络。
- `src/unnp_denoise/train.py`：训练和早停记录。
- `src/unnp_denoise/metrics.py`：PSNR、SSIM。
- `src/unnp_denoise/baseline.py`、`src/unnp_denoise/bm3d_worker.py`：BM3D 对比方法。
- `src/unnp_denoise/visualize.py`：保存图片、曲线和对比图。

## 方法边界

BM3D 的效果通常强于这个小型 UNNP 实现，这是合理现象：BM3D 是成熟传统算法，而这里的 UNNP 版本为了能在 CPU 上完整跑通，使用了小尺寸图片和较紧凑的网络。项目重点是展示 UNNP 的核心机制：只用单张带噪图训练、通过早停避免拟合噪声，并用曲线和指标证明这一点。
